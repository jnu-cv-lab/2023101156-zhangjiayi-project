# 实验 README：基于骨架序列与 Transformer 的羽毛球击球动作识别

## 一、实验背景
羽毛球击球动作具有明显的时间动态特征，传统基于原始视频的分类方法需要大量标注数据和较高 GPU 显存。本实验采用 MediaPipe Pose 将视频转换为人体骨架时间序列，再用轻量级 Transformer Encoder 完成动作分类。该方法计算量低、可解释性强，适合课堂环境下完成训练、测试与推理全流程。

## 二、数据集
使用 Kaggle 公开数据集 **badminton_stroke_video**，包含 6 类羽毛球击球动作：
- forehand drive（正手平抽）
- forehand lift（正手挑球）
- forehand net shot（正手网前球）
- forehand clear（正手高远球）
- backhand drive（反手平抽）
- backhand net shot（反手网前球）
数据集以 .mp4 视频片段形式提供，实验中共提取有效样本 771 个，按 8:2 划分为训练集（616 个）和测试集（155 个）。

## 三、方法流程
1. **视频预处理**：使用 OpenCV 逐帧读取视频。
2. **骨架提取**：利用 MediaPipe Pose 对每一帧提取 33 个人体关键点的 `(x, y, z, visibility)`，每帧展平为 132 维向量。
3. **重采样**：将所有视频的帧序列统一线性插值重采样为固定 30 帧（`T=30`），输入形状为 `(30, 132)`。
4. **姿态归一化**：以左右髋部中心为原点，以肩宽为尺度进行平移和缩放，消除人物体型与拍摄距离的影响。
5. **数据集划分**：随机分割为训练集（80%）和测试集（20%），保存为 `.npy` 文件（`X_train.npy`, `y_train.npy`, `X_test.npy`, `y_test.npy`）。

## 四、模型结构
采用 **Transformer Encoder** 作为分类骨架，具体参数如下：
| 组件               | 设置值                     |
|------------------|---------------------------|
| 输入维度           | `[B, 30, 132]`            |
| 线性嵌入层         | 132 → 128 (`d_model`)     |
| 位置编码           | 可学习绝对位置编码         |
| Transformer 层数  | 2 (`num_layers`)          |
| 注意力头数         | 4 (`nhead`)               |
| 前馈网络维度       | 256 (`dim_feedforward`)   |
| Dropout           | 0.1                       |
| 池化方式           | 时间维度平均池化           |
| 分类层             | 128 → 6 (输出类别数)      |
模型总参数量约 50 万，适合 CPU 环境快速训练。

## 五、训练与测试
- **损失函数**：交叉熵损失（`CrossEntropyLoss`）
- **优化器**：Adam（学习率 `1e-3`）
- **批大小**：16
- **训练轮数**：20
**训练曲线**（简要）：
- 训练损失从 1.87 下降至 0.99，训练准确率从 20% 提升至 62%。
- 测试准确率在 30%~42% 之间波动，最终达到 **41.94%**（高于随机猜测 16.7%）。
**测试结果**：
- 整体测试准确率约 42%。
- 混淆矩阵显示模型对正手挑球与正手网前球、正手高远球与反手平抽等动作存在混淆。
- 单样本推理（`forehand clear/002.mp4`）预测正确，置信度 0.7255；而对反手平抽视频曾误判为正手高远球。
## 六、问题分析与改进方向
### 存在问题
1. **准确率偏低**：测试准确率仅 42%，远未达到实用水平。
2. **过拟合**：训练准确率（62%）明显高于测试准确率（42%），模型未能泛化。
3. **左右手混淆**：未对左利手视频进行镜像对齐，导致正手/反手动作判别困难。
4. **数据不足**：616 个训练视频对于 Transformer 模型偏少。
### 改进建议
- **数据增强**：时间轴抖动、随机缩放、关键点噪声、镜像翻转统一为相同握拍侧。
- **模型调整**：增加 Transformer 层数至 4 或改用 LSTM；提高 Dropout 至 0.3。
- **训练策略**：增加 epoch 至 100，使用学习率调度（`ReduceLROnPlateau`）和早停。
- **归一化验证**：确保髋部中心和肩宽缩放生效，打印骨架数值范围应落在 `[-1, 1]` 左右。

## 七、实验结论
本实验成功实现了从羽毛球视频到人体骨架、再到 Transformer 分类的端到端流程。尽管当前模型测试准确率不高（42%），但验证了骨架序列化 + Transformer 在动作识别任务中的可行性。通过数据增强、镜像对齐和模型加深等改进，有望显著提升分类性能。实验代码完整，包含预处理、训练、测试和推理四个模块，可作为进一步研究的基础。
